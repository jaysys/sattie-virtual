# 데이터 사전

## 1. 문서 목적
- 마크다운 뷰어 웹서비스에서 사용하는 핵심 데이터 필드의 의미, 타입, 기본값, 생성 규칙을 고정한다.
- 구현팀이 모델 정의, 인덱스 생성, API 응답을 일관되게 맞추도록 기준을 제공한다.

## 2. 공통 규칙
- 날짜/시간은 ISO 8601 UTC 문자열을 사용한다.
- URL 필드는 상대경로를 기본으로 한다.
- 문자열 필드는 별도 명시가 없으면 trim 후 저장한다.
- 배열 필드는 별도 명시가 없으면 빈 배열을 기본값으로 사용한다.
- 불리언 필드는 별도 명시가 없으면 `false` 또는 문맥상 안전한 기본값을 사용한다.

## 3. 문서 메타데이터 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 | 생성 규칙 |
|---|---|---|---|---|---|
| `slug` | string | Y | 없음 | 외부 공개 식별자 | 파일명 또는 제목 기반 생성, 충돌 시 suffix 추가 |
| `title` | string | Y | 파일명 또는 첫 H1 | 문서 제목 | front matter 우선, 없으면 본문 추출 |
| `summary` | string | Y | 첫 문단 요약 | 문서 한 줄 설명 | front matter 우선, 없으면 본문 첫 문단 |
| `source_path` | string | Y | 없음 | 내부 원본 파일 경로 | 스캔된 상대경로 저장 |
| `public_url` | string | Y | 없음 | 외부 문서 접근 경로 | `/p/{slug}` |
| `category` | string | Y | `general` | 문서 카테고리 | front matter 우선, 없으면 경로 규칙 |
| `tags` | string[] | Y | `[]` | 문서 태그 | front matter 우선 |
| `is_public` | boolean | Y | `true` | 허브 노출 여부 | front matter 우선 |
| `updated_at` | string | Y | 파일 수정시각 | 최신 수정 시각 | 파일 메타데이터 또는 생성 시각 |
| `word_count` | integer | Y | `0` | 본문 단어 수 | Markdown 평문 기준 계산 |
| `reading_minutes` | integer | Y | `1` | 예상 읽기 시간 | `word_count / 250` 올림, 최소 1 |
| `headings` | array | Y | `[]` | 목차용 헤더 목록 | H2~H3 추출 |
| `order` | integer | N | `0` | 카테고리 내 정렬 우선순위 | front matter 우선 |

## 4. Heading 객체 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 | 생성 규칙 |
|---|---|---|---|---|---|
| `level` | integer | Y | 없음 | 제목 레벨 | H2=2, H3=3 |
| `text` | string | Y | 없음 | 헤더 텍스트 | Markdown 헤더에서 추출 |
| `anchor` | string | Y | 없음 | 섹션 링크용 id | 텍스트 slugify |

## 5. 검색 결과 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 | 생성 규칙 |
|---|---|---|---|---|---|
| `score` | number | Y | `0` | 검색 적합도 점수 | 제목/태그/요약/본문 가중치 합산 |
| `matched_fields` | string[] | Y | `[]` | 일치한 필드 목록 | 검색 엔진 결과 기반 |

## 6. 업로드 요청 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 | 검증 규칙 |
|---|---|---|---|---|---|
| `filename` | string | Y | 없음 | 업로드 파일명 | `.md` 확장자만 허용 |
| `content` | string | Y | 없음 | Markdown 본문 | 최대 용량 제한 적용 |

## 7. 업로드 응답 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 | 생성 규칙 |
|---|---|---|---|---|---|
| `ok` | boolean | Y | `true` | 업로드 성공 여부 | 성공 응답 고정값 |
| `file_name` | string | Y | 없음 | 실제 저장 파일명 | sanitize 후 저장 결과 |
| `size_bytes` | integer | Y | 없음 | 저장 바이트 수 | UTF-8 기준 |
| `slug` | string | Y | 없음 | 생성된 문서 slug | 인덱서 생성 결과 |
| `public_url` | string | Y | 없음 | 문서 상세 경로 | `/p/{slug}` |
| `reindexed` | boolean | Y | 없음 | 재색인 성공 여부 | 업로드 후 인덱스 반영 결과 |

## 8. 목록 응답 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 |
|---|---|---|---|---|
| `ok` | boolean | Y | `true` | 성공 여부 |
| `count` | integer | Y | `0` | 전체 결과 수 |
| `page` | integer | Y | `1` | 현재 페이지 |
| `page_size` | integer | Y | `20` | 페이지 크기 |
| `items` | array | Y | `[]` | 문서 요약 배열 |

## 9. 상세 응답 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 |
|---|---|---|---|---|
| `ok` | boolean | Y | `true` | 성공 여부 |
| `document` | object | Y | 없음 | 문서 상세 객체 |
| `related_documents` | array | Y | `[]` | 관련 문서 목록 |

## 10. 에러 응답 필드 정의

| 필드명 | 타입 | 필수 | 기본값 | 설명 |
|---|---|---|---|---|
| `ok` | boolean | Y | `false` | 실패 여부 |
| `error_code` | string | Y | 없음 | 기계 처리용 오류 코드 |
| `message` | string | Y | 없음 | 사용자 표시용 오류 메시지 |

## 11. 오류 코드 사전

| 코드 | HTTP 상태 | 설명 |
|---|---|---|
| `DOC_NOT_FOUND` | 404 | slug 또는 문서 경로를 찾을 수 없음 |
| `INVALID_UPLOAD_TYPE` | 400 | `.md` 외 파일 업로드 |
| `INVALID_FILENAME` | 400 | 허용되지 않는 파일명 |
| `EMPTY_QUERY` | 400 | 검색어 없음 |
| `INVALID_SORT` | 400 | 허용되지 않는 정렬 파라미터 |
| `INDEX_BUILD_FAILED` | 500 | 인덱스 재생성 실패 |
| `SEARCH_INDEX_FAILED` | 500 | 검색 인덱스 조회 실패 |
| `RENDER_FAILED` | 500 | HTML 렌더링 실패 |

## 12. 파생 규칙
- `public_url = "/p/" + slug`
- `raw_markdown_url = "/api/docs/" + slug + "/raw"`
- `reading_minutes = max(1, ceil(word_count / 250))`
- `category`가 비어 있으면 `general`
- `tags`가 문자열 하나로 입력되면 내부적으로 배열로 정규화

## 13. 구현 메모
- 모델 정의는 데이터 사전과 정확히 일치해야 한다.
- JSON 인덱스 파일 구조는 이 문서를 기준으로 고정한다.
- 필드 추가가 필요하면 설계 문서보다 먼저 데이터 사전을 갱신해야 한다.
