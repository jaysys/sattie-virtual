# QA 체크리스트

## 1. 문서 목적
- 출시 전 검수해야 할 항목을 운영 관점에서 고정한다.
- 기능 동작 여부뿐 아니라 사용자 경험, 오류 처리, 회귀 여부까지 점검한다.

## 2. 검수 원칙
- 체크리스트는 실제 브라우저 사용 흐름 기준으로 검수한다.
- 한 항목이라도 핵심 경로를 막으면 출시 보류 대상이다.
- 기존 위성영상 데모 회귀는 문서 서비스와 별개로 반드시 확인한다.

## 3. 랜딩 페이지 검수
- 서비스 핵심 메시지가 첫 화면에서 보이는가
- 허브 이동 버튼이 정상 동작하는가
- 업로드 진입 버튼이 정상 동작하는가
- 샘플 문서 유입 경로가 명확한가
- 모바일 화면에서 CTA 버튼이 잘 보이는가

## 4. 문서 허브 검수
- 문서 목록이 정상 노출되는가
- 카테고리 필터가 정상 동작하는가
- 태그 필터가 정상 동작하는가
- 정렬이 정상 동작하는가
- 검색창이 정상 동작하는가
- 검색 결과 없음 상태가 이해 가능하게 보이는가
- 문서 카드에서 제목, 요약, 카테고리, 읽기 시간이 자연스럽게 보이는가

## 5. 문서 상세 검수
- slug URL로 직접 접근 가능한가
- 제목과 요약이 정상 표시되는가
- 본문이 깨지지 않고 렌더링되는가
- Mermaid가 정상 표시되는가
- MathJax가 정상 표시되는가
- 표가 레이아웃을 깨뜨리지 않는가
- 이미지 클릭 확대가 동작하는가
- TOC 클릭 시 해당 섹션으로 이동하는가
- heading 링크 복사 기능이 정상 동작하는가
- 코드블록 복사 기능이 정상 동작하는가
- 관련 문서가 노출되고 이동 가능한가
- 원본 Markdown 다운로드 또는 조회가 가능한가

## 6. 업로드 검수
- `.md` 파일 업로드가 정상 동작하는가
- `.md` 이외 파일이 차단되는가
- 잘못된 파일명 입력이 차단되는가
- 업로드 중 상태 메시지가 보이는가
- 업로드 성공 후 상세 페이지 링크가 제공되는가
- 업로드 직후 검색과 허브 목록에 반영되는가
- 실패 시 원인과 재시도 경로가 보이는가

## 7. 검색 검수
- 제목 일치 문서가 우선 노출되는가
- 태그 일치가 제목 다음 우선순위로 보이는가
- 요약/본문 일치도 결과에 반영되는가
- 검색어가 비었을 때 적절한 안내가 보이는가
- 특수문자 입력 시 오류 없이 처리되는가

## 8. 운영자 화면 검수
- 총 문서 수가 정상 표시되는가
- 공개 상태가 정상 표시되는가
- 재색인 버튼이 정상 동작하는가
- 재색인 성공/실패 피드백이 분명한가
- 인덱스 실패 후에도 기존 읽기 서비스가 유지되는가
- 운영 화면이 외부 공개 사용자에게 무방비로 노출되지 않는가

## 9. 오류 처리 검수
- 없는 slug 접근 시 404 안내가 자연스러운가
- 렌더링 실패 시 원문 다운로드 경로가 제공되는가
- 잘못된 검색 요청 시 의미 있는 메시지가 보이는가
- 업로드 오류 시 사용자에게 이해 가능한 안내가 보이는가
- 서버 내부 오류가 그대로 노출되지 않는가

## 10. 반응형 검수
- 데스크톱에서 3단 레이아웃이 정상 동작하는가
- 태블릿에서 TOC와 본문이 충돌하지 않는가
- 모바일에서 본문 가독성이 유지되는가
- 모바일에서 메뉴/TOC/공유 UI가 겹치지 않는가
- 긴 코드블록과 표가 화면 밖으로 과도하게 깨지지 않는가

## 11. 접근성 검수
- 키보드만으로 검색, 이동, 업로드가 가능한가
- 버튼과 링크에 의미 있는 라벨이 있는가
- 포커스 상태가 눈에 띄는가
- 색상만으로 상태를 구분하지 않는가
- heading 계층이 문서 구조와 일치하는가

## 12. SEO 및 공유 검수
- 문서별 `<title>`이 적절한가
- description 메타가 생성되는가
- canonical URL이 올바른가
- Open Graph 메타가 들어가는가
- 공유 시 제목과 요약이 자연스럽게 보이는가
- 헤더 앵커 URL이 직접 접근 가능한가

## 13. 데이터 및 인덱스 검수
- `documents.json`이 정상 생성되는가
- `search.json`이 정상 생성되는가
- front matter가 있는 문서와 없는 문서 모두 인덱싱되는가
- `guide/`, `docs/`, `uploads/docs/` 문서가 모두 규칙대로 스캔되는가
- slug 충돌 시 충돌이 안전하게 해소되는가
- 카테고리와 태그 기본값이 규칙대로 들어가는가

## 14. 회귀 검수
- 기존 `/guide` 목록 조회가 정상 동작하는가
- 기존 `/guide/{path}` 렌더링이 정상 동작하는가
- 기존 `/guide/upload`가 깨지지 않았는가
- 기존 `/ui`가 정상 동작하는가
- 기존 위성영상 API가 영향받지 않았는가

## 15. 출시 차단 조건
- 허브 접근 불가
- 상세 페이지 렌더링 불가
- 업로드 후 재색인 실패로 문서가 노출되지 않음
- 검색이 핵심 문서를 찾지 못함
- 모바일에서 본문 읽기가 어려움
- `/guide` 또는 `/ui` 회귀 발생

## 16. 최종 확인 기준
- 핵심 사용자 흐름 3개가 모두 성공해야 한다.
  - 허브 탐색
  - 상세 읽기
  - 업로드 후 즉시 열람
- 운영자 흐름 2개가 모두 성공해야 한다.
  - 재색인
  - 오류 확인 및 복구
- 회귀 검수 100% 통과 전에는 출시하지 않는다.
