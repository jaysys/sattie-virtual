# 설계 문서 패키지 안내

## 문서 구성
- `마크다운_뷰어_웹서비스_설계서.md`
  - 전체 서비스 구조, 범위, 라우트, 데이터, 보안, 성능, 출시 기준을 담은 메인 설계서
- `API_응답_명세_부록.md`
  - 구현팀이 그대로 사용할 수 있는 JSON API 응답 명세 부록
- `화면별_와이어프레임_컴포넌트_설계서.md`
  - 랜딩, 허브, 상세, 업로드, 운영 화면의 구조와 컴포넌트 경계를 정의한 화면 설계서
- `개발_인수인계_명세서.md`
  - 구현 순서, Epic, Task, 완료 기준을 담은 개발 인수인계 문서
- `상세_인터랙션_상태전이_설계서.md`
  - 로딩, 성공, 실패, 빈 상태, 재시도 흐름을 정의한 인터랙션 설계서
- `데이터_사전.md`
  - 필드 타입, 기본값, 생성 규칙, 오류 코드까지 고정한 데이터 규약 문서
- `QA_체크리스트.md`
  - 출시 전 운영 관점 검수 항목을 정리한 QA 기준 문서

## 사용 목적
- 메인 설계서는 전체 구조와 우선순위 판단 기준으로 사용한다.
- API 부록은 백엔드 구현 기준으로 사용한다.
- 화면 설계서는 프론트엔드 구현 기준으로 사용한다.

- 구현팀 인수인계 가능 수준
