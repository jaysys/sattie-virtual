# 상세 인터랙션 상태전이 설계서

## 1. 문서 목적
- 핵심 화면과 주요 기능의 상태 전이와 사용자 인터랙션을 정의한다.
- 구현팀이 로딩, 성공, 실패, 빈 상태를 일관되게 처리하도록 기준을 제공한다.

## 2. 적용 대상
- 랜딩 페이지
- 문서 허브 페이지
- 문서 상세 페이지
- 업로드 페이지
- 운영자 문서 관리 페이지

## 3. 상태 설계 원칙
- 모든 주요 기능은 `idle`, `loading`, `success`, `empty`, `error` 중 하나의 상태를 가져야 한다.
- 사용자 액션 이후에는 반드시 시각적 피드백이 있어야 한다.
- 실패 상태는 재시도 경로를 함께 제공해야 한다.

## 4. 문서 허브 상태 전이

### 4.1 기본 상태
- `idle`
  - 첫 진입 전 또는 초기 렌더 준비 상태
- `loading`
  - 목록/카테고리/검색 데이터 로딩 중
- `success`
  - 문서 목록 표시 가능
- `empty`
  - 문서가 없거나 필터 결과 없음
- `error`
  - API 실패 또는 인덱스 읽기 실패

### 4.2 상태 전이
```text
idle -> loading -> success
loading -> empty
loading -> error
success -> loading
empty -> loading
error -> loading
```

### 4.3 사용자 액션
- 검색어 입력
  - 기존 상태와 무관하게 `loading` 전환 후 결과 반영
- 카테고리 선택
  - `loading` 전환 후 필터 결과 갱신
- 정렬 변경
  - `loading` 전환 후 리스트 재배열
- 페이지 이동
  - `loading` 전환 후 다음 페이지 조회

### 4.4 빈 상태 규칙
- 전체 문서가 없을 때
  - "첫 문서를 업로드하세요" 메시지 표시
- 검색 결과가 없을 때
  - 검색어 표시
  - 필터 초기화 버튼 표시

## 5. 문서 상세 상태 전이

### 5.1 기본 상태
- `loading`
  - slug 기반 문서 조회 중
- `success`
  - 문서 본문 렌더 완료
- `not_found`
  - slug에 해당 문서 없음
- `render_error`
  - 문서는 존재하지만 렌더링 실패

### 5.2 상태 전이
```text
loading -> success
loading -> not_found
loading -> render_error
success -> loading
```

### 5.3 상세 인터랙션
- TOC 클릭
  - 해당 anchor로 이동
  - 현재 섹션 강조
- heading 링크 클릭
  - anchor URL 복사 또는 해당 섹션 위치 갱신
- code copy 클릭
  - 복사 성공 토스트 표시
- raw markdown 클릭
  - 새 창 또는 다운로드로 원문 제공
- 관련 문서 클릭
  - 현재 상세 상태 종료 후 새 slug로 `loading`

### 5.4 오류 상태 규칙
- `not_found`
  - 404 안내
  - 허브로 돌아가기 버튼
- `render_error`
  - 문서 렌더 실패 안내
  - 원문 다운로드 제공
  - 재시도 또는 허브 이동 제공

## 6. 업로드 상태 전이

### 6.1 기본 상태
- `idle`
  - 파일 선택 전
- `file_selected`
  - 유효한 `.md` 선택됨
- `uploading`
  - 업로드 전송 중
- `success`
  - 저장 및 재색인 성공
- `validation_error`
  - 확장자 또는 파일명 문제
- `upload_error`
  - 업로드 실패 또는 재색인 실패

### 6.2 상태 전이
```text
idle -> file_selected
file_selected -> uploading
uploading -> success
uploading -> validation_error
uploading -> upload_error
validation_error -> file_selected
upload_error -> file_selected
```

### 6.3 상태별 UI 규칙
- `idle`
  - 파일 선택 버튼만 강조
- `file_selected`
  - 파일명 표시
  - 업로드 버튼 활성화
- `uploading`
  - 버튼 비활성화
  - 진행 메시지 표시
- `success`
  - 상세 페이지 이동 링크 표시
  - 자동 이동 가능
- `validation_error`
  - 확장자 제한 및 파일명 규칙 노출
- `upload_error`
  - 재시도 버튼 표시

## 7. 운영자 문서 관리 상태 전이

### 7.1 기본 상태
- `loading`
  - 문서 목록/통계 로딩 중
- `success`
  - 표와 요약 카드 표시
- `empty`
  - 문서 없음
- `error`
  - 인덱스 조회 실패
- `reindexing`
  - 전체 재색인 수행 중
- `reindex_success`
  - 재색인 완료
- `reindex_error`
  - 재색인 실패

### 7.2 상태 전이
```text
loading -> success
loading -> empty
loading -> error
success -> reindexing
reindexing -> reindex_success
reindexing -> reindex_error
reindex_success -> success
reindex_error -> success
```

### 7.3 운영 액션
- 새로고침
  - `loading` 전환
- 전체 재색인
  - `reindexing` 전환
- 상태 필터
  - `loading` 전환 후 표 갱신

### 7.4 운영 화면 범위 규칙
- MVP 운영 화면은 인덱스 상태 조회와 재색인만 담당한다.
- 문서 공개 여부 변경, 카테고리 수정, 태그 수정은 MVP 범위에서 제외한다.
- 표의 상태 값은 front matter의 `is_public` 기준으로 읽기 전용 표시만 한다.

## 8. 검색 인터랙션 설계

### 8.1 입력 상태
- `idle`
  - 입력 전
- `typing`
  - 입력 중
- `loading`
  - 검색 요청 진행 중
- `success`
  - 결과 존재
- `empty`
  - 결과 없음
- `error`
  - 검색 실패

### 8.2 검색 UX 규칙
- 너무 짧은 검색어는 요청을 지연하거나 제한할 수 있다.
- 엔터 입력 또는 디바운스 후 검색 요청
- 검색 결과 수를 명시한다.
- 검색 실패 시 기존 성공 결과를 즉시 지우지 않고, 에러 상태 메시지를 별도 노출할 수 있다.

## 9. 공유 인터랙션 설계

### 9.1 액션 종류
- 현재 문서 링크 복사
- 현재 heading 링크 복사
- 원문 링크 복사

### 9.2 상태
- `idle`
- `copied`
- `error`

### 9.3 UX 규칙
- 복사 성공 시 짧은 피드백 제공
- 실패 시 수동 복사 가능한 URL을 보여줄 수 있어야 한다.

## 10. 반응형 상태 처리
- 모바일에서 사이드바는 기본적으로 닫힘 상태
- TOC 드로어는 문서 스크롤 중 본문을 가리지 않아야 한다.
- 업로드 성공 후 자동 이동은 모바일 브라우저 전환을 방해하지 않아야 한다.

## 11. 공통 오류 메시지 원칙
- 사용자 언어로 간결하게 보여준다.
- 기술적 원인은 내부 로그나 관리자 화면에서 확인 가능하게 분리한다.
- 예시
  - "문서를 찾을 수 없습니다."
  - "업로드 가능한 파일 형식은 .md 입니다."
  - "문서를 불러오는 중 문제가 발생했습니다. 다시 시도하세요."

## 12. 최종 구현 기준
- 모든 핵심 화면은 상태 전이 표 없이도 예측 가능하게 동작해야 한다.
- 사용자는 현재 무엇이 진행 중인지 항상 알 수 있어야 한다.
- 실패 상태는 끝 상태가 아니라 재시도 가능한 상태여야 한다.
